Timetable Generation Project

Overview:

Our team developed a comprehensive timetable generation project using Next.js for the frontend, Python Django for the backend, and MySQL for the database. The system automates the creation of timetables for both students and staff, significantly reducing manual effort and ensuring optimal allocation of classes and resources. It features different access levels: Director, Principal, HOD, Tutor, and Faculty. Each level has decreasing access rights, with the Director having the most control and the Faculty level being limited to viewing only their subject timetable. For example, tutors can view all faculty schedules for their assigned classes. The generated timetables can be downloaded in PDF format, offering a convenient and user-friendly solution for all users.

Key Features:

  • User-Friendly Interface:
  • Role-Based Access Control:
    • Director Level: Full access to all timetables and administrative functions.
    • Principal Level: Access to most functions, with slight restrictions compared to the Director.
    • HOD Level: Access to department-specific timetables and faculty schedules.
    • Tutor Level: Access to all faculty schedules for their assigned classes.
    • Faculty Level: Access to view only their subject timetables.
  • PDF Download

Technologies Used:

  • Backend: Django Framework
  • Frontend: Next JS
  • Database: MySQL
